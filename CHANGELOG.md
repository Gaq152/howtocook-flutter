# Changelog

本文件记录 HowToCook 的所有发布变更。

格式遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

> 客户端检查到新版本时，更新弹窗会直接渲染这里对应版本的条目，请保持 Markdown 可读。

## [Unreleased]

## [0.3.3] - 2026-05-07

### 新增
- **封面与详情图分离编辑**：菜谱编辑/创建页面将图片拆分为「封面图片」和「详情图片」两个独立区域，操作更清晰。
- **更新日志本地化**：更新日志改为读取本地 CHANGELOG.md，新增「当前版本」和「全部版本」标签页切换；移除关于页的开源仓库卡片。
- **分享卡片工具板块**：分享图片和二维码数据中增加所需工具信息展示。

### 变更
- **DeepSeek V4 升级**：内置模型从 deepseek-chat 升级到 deepseek-v4-flash（100 万上下文），能力数据库新增 V4 Flash / V4 Pro。
- **模型管理按钮位置**：「添加模型」按钮从右下角浮动按钮移至标题栏加号图标，风格统一。
- **分类名称统一**：创建菜谱和扫码解析的分类名称改为从 manifest 动态获取，补全调料、半成品分类。
- **二维码数据优化**：移除冗余字段，列表改用换行符分隔，体积减小约 15-25%。

### 修复
- **详情图不显示**：修复已下载的详情图始终显示占位图的问题（assets/images/ 路径被错误识别为内置资源）。
- **步骤标题未解析**：步骤中的 Markdown 标题（如 `### 准备工作`）现在渲染为分组标题，不再原样显示。
- **扫码无响应**：新增定时 WeChatQRCode 独立扫描，解决 MLKit 对高密度码完全不触发回调的问题；扫描页面增加实时状态提示。
- **二维码可扫描性**：分享卡片二维码改用低纠错级别、加大安静区、黑白高对比度边框，提升扫码成功率。
- **检查更新失效**：split-per-abi 构建导致 APK versionCode 带 ABI 前缀（arm64=2000+N），与 manifest 不一致导致永远检测不到更新；现已修复比较逻辑并同步 CI manifest。
- **版本号显示异常**：关于页和设置页不再显示带 ABI 前缀的 buildNumber，改为只显示版本号。

## [0.3.2] - 2026-04-23

### 修复
- **后台更新通知**：申请 Android 13+ 通知权限，修复通知栏进度条不显示问题。
- **通知栏操作**：暂停、继续、取消按钮现可直接在通知栏响应。
- **安装失败提示**：下载完成后安装失败不再静默，弹窗显示具体错误。
- **去除应用内横幅**：移除顶部下载进度横幅，仅保留通知栏进度条。

## [0.3.1] - 2026-04-22

### 新增
- **更新分包**：按 CPU 架构分别构建 arm64 / arm32 APK，包体从 160MB 降至约 55MB，自动更新按设备 ABI 选择对应包下载。

### 变更
- **创作文案统一**：「创建菜谱」「新建菜谱」「新增教程」等入口统一改为「创作菜谱」「创作教程」；「我的自创」改为「我的创作」。

### 修复
- **AppBar 标题不可见**：教程中心、收藏、创作等页面标题颜色与背景融合，显式设置浅色/深色主题各自的标题颜色。
- **相机扫码无响应**：改为 MLKit 定位到二维码后再交由 WeChatQRCode 解码，解决高密度码实时扫描失败问题；WeChatQRCode 提前初始化，降低首次触发延迟。
- **收藏/自创数量统计不全**：「我的」页面三项统计数字合并菜谱与教程两个来源。
- **详情页显示封面图**：详情页头部改回直接显示详情图，不再错误地走封面图降级逻辑。
- **卡片不显示用户上传图片**：菜谱卡片优先展示用户上传的本地/网络图片。

## [0.3.0] - 2026-04-22

### 新增
- **关于页面**：设置页「关于」入口改为独立页面，包含开发者信息、开源仓库、更新日志、问题反馈与第三方许可证入口。
- **后台下载更新**：点击「立即更新」后支持后台下载 APK，通知栏实时显示进度条，支持暂停与取消操作。
- **全局下载横幅**：下载进行中或暂停时，应用顶部显示进度横幅，点击可打开更新弹窗。
- **更新日志入口**：设置页「关于」栏新增「更新日志」选项，可查看当前版本更新内容。

### 变更
- **更新弹窗重构**：移除「跳过本版」「稍后提醒」按钮，改为「取消 / 后台下载 / 立即更新」，下载中支持暂停/继续/取消。
- **教程详情**：内容改用 MarkdownBody 渲染，正确显示标题、代码块、加粗等 Markdown 格式。

### 修复
- **用户上传图片不显示**：菜谱详情页正确识别本地路径与 base64 图片并直接渲染。
- **多图轮播手势失效**：详情页多图模式下，用户手动滑动时暂停自动滚动，松手后恢复。
- **封面图降级逻辑**：封面图不存在时依次回退至详情图第一张，再回退至通用占位图。
- **SnackBar 遮挡导航栏**：封装 AppSnackBar，统一为底部胶囊导航栏和操作区留出安全间距。

## [0.2.0] - 2026-04-19

### 变更
- **全局主题重构**：色系从橙色系改为温暖治愈系奶油米色，主色调改为陶土色（#D06A4C），辅助色改为鼠尾草绿（#8CA471），新增李子色与黄油色装饰色。
- **深色模式**：配色同步更新为暖棕色系，视觉更柔和。
- **卡片与按钮**：圆角从 12/8 增大至 20/16，elevation 归零，改用带透明度主色调阴影，整体更扁平轻盈。
- **底部导航栏**：重构样式适配新主题，去除多余阴影。
- **菜谱模块**：首页、详情页、卡片、创建/编辑、搜索、分享卡片全面适配新主题，层次与可读性提升。
- **AI 聊天**：标题改为「小厨」并左对齐，修复键盘弹出时布局挤压，空状态图标更新。
- **教程模块**：列表页、详情页、编辑页、分享卡片全面适配新主题。
- **用户中心**：页面布局大幅优化，个人信息与功能入口视觉重构。
- **设置模块**：模型管理页、设置页、更新弹窗适配新色系。
- **CI/CD**：release.yml 改为直接写入 .env，精简内置 API 配置，限流默认值改为 10。
- **数据同步**：移除 dotenv 依赖，静态资源 base URL 硬编码。
- **教程分类顺序**：manifest.json 中「基础知识」置于「进阶知识」之前。

## [0.1.1] - 2026-04-18

### 新增
- **模型管理页免责声明**：底部新增小字提示，说明自定义模型输出质量取决于服务商。

### 变更
- **「我的」页精简入口**：模型管理、数据同步、关于 三项从主页移除，统一收敛到「设置」页入口，减少重复。
- **底部导航栏重构**：移除突出 FAB 设计，改为标准三项导航（菜谱 / AI / 我的），解决 FAB 遮挡输入框问题。
- **AI 提示词优化**：工具调用规则改为"自然语言整理呈现"，避免模型直接输出原始 JSON。

### 修复
- **DeepSeek / OpenAI 工具调用**：`sendMessageSync` 改为直接解析 SSE 流，修复 tool_calls 被纯文本流丢弃导致 MCP 工具无法触发的问题。
- **AI 多轮工具调用文本保留**：修复 AI 先输出文字再调用工具时，前置文本被"正在调用工具..."覆盖丢失的问题。
- **菜谱工具 Chip 字体不可见**：显式指定 labelStyle 颜色，修复浅色背景下文字与背景色接近导致看不清的问题。

## [0.1.0] - 2026-04-18

首个正式公开版本。

### 新增
- **菜谱管理**：列表 / 搜索 / 筛选 / 收藏 / 笔记 / 编辑 / 创建，支持二维码分享与扫码导入。
- **AI 助手**：多模型对话（Claude / GPT / DeepSeek），图片识别，流式输出，思考链实时展示。
- **MCP 工具**：内置 `create_recipe`，AI 可在对话中直接生成菜谱预览卡片。
- **模型管理**：设置页支持自定义 API Key，内置 Key 与自定义 Key 一键切换。
- **数据同步**：内置资源打包 + GitHub Pages 增量更新，封面图与详情图可独立开关下载。
- **教程系统**：查看 / 创建 / 编辑烹饪教程，按分类组织。
- **自动更新**：启动 3 秒后静默检查 GitHub Releases，支持 ghfast.top 镜像回退，SHA256 校验，断点缓存复用。
- **用户中心**：我的收藏、我的自创。

### 变更
- 应用包名统一为 `com.anlife.howtocook`。
- Release 构建启用独立 keystore 签名，CI 与本地通过 `key.properties` 共享同一密钥。
- 新增 `REQUEST_INSTALL_PACKAGES` 权限以支持应用内更新安装。

### 修复
- 解决 AGP 8 下部分三方插件（如 `install_plugin`）缺失 `namespace` 导致的构建失败。
